# Security Policy

## Supported Versions

Autograph is in early development (pre-1.0). Security fixes are made against the
latest released version only; there is no backport policy yet.

## Reporting a Vulnerability

Please do **not** open a public GitHub issue for security vulnerabilities.

Instead, report it privately via
[GitHub Security Advisories](https://github.com/uny/autograph/security/advisories/new)
for this repository. Include:

- A description of the vulnerability and its potential impact
- Steps to reproduce (a minimal repro project or snippet helps)
- Affected version(s)

You should expect an initial response within a few days. Once a fix is available,
we'll coordinate disclosure and credit with you before making the advisory public.
